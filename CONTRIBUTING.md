This guide will help you get started quickly and contribute effectively.

## How to Get Started

1. Fork the Repository: Click the **Fork** button at the top right of the repo page.
2. Clone Your Fork: Replace `<your-username>` with your GitHub username:
   git clone https://github.com/<your-username>/sweetescape-music-player.git
3. Create a New Branch:
   git checkout -b feature/my-new-feature
4. Run the Project: This is a static web project — no backend required.  
   Open **index.html** in your browser, or use VS Code Live Server for auto-refresh.

## Issue Labels

- **good-first-issue** — Beginner-friendly tasks
- **help-wanted** — Actively needed contributions
- **enhancement** — New features or improvements
- **bug** — Problems or broken behavior
- **ui** — Interface or visual improvements
- **documentation** — Docs, comments, or guides

---

## Pull Request Guidelines

1. Keep PRs focused on **one change** only.
2. Double check you didn't break anything
3. Clearly describe what you changed and why(optional).
4. Link an Issue (example: `Closes #12`) when relevant.
5. Test your changes locally before submitting.
6. You do not need to keep code clean, readable, and consistent.
